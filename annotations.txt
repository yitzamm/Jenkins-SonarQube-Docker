Jenkins-SonarQube-Docker

CREATE THE EC2 INSTANCES

Create 3 EC2 instances in the AWS console for Jenkins, SonarQube and Docker respectively
Create an SSH key pair to access the VM remotely
NOTE: Use the t2-medium instance size

SET-UP JENKINS

Jenkins Installation Guide: https://www.jenkins.io/doc/book/installing/linux/#debianubuntu

Go to WSL:

cd ~/Downloads/ *
chmod 400 ssh-key-pair.pem (grant read-only access to the file owner only)
ssh -i ssh-key-pair.pem ubuntu@jenkins_EC2_public_IP (ubuntu@ = default user)

Java:
sudo apt update
sudo apt install fontconfig openjdk-21-jre
java -version

Jenkins LTS:
sudo wget -O /etc/apt/keyrings/jenkins-keyring.asc \
  https://pkg.jenkins.io/debian-stable/jenkins.io-2026.key
echo "deb [signed-by=/etc/apt/keyrings/jenkins-keyring.asc]" \
  https://pkg.jenkins.io/debian-stable binary/ | sudo tee \
  /etc/apt/sources.list.d/jenkins.list > /dev/null
sudo apt update
sudo apt install Jenkins

Verify installation with: systemctl status Jenkins
Open port 8080 for Jenkins' EC2 instance

CREATE A NEW PROJECT ON THE JENKINS SERVER

Gather Jenkins' access key: sudo cat /var/lib/jenkins/secrets/initialAdminPassword
Create a free-style project and enable GitHub hook trigger for GITSCM polling
On GitHub go to Settings > WebHooks and use the instance IP+port in the payload URL field: http://52.201.245.10:8080/github-webhook/
Additionally, use the select individual events also and check both Pull and Push requests
Create a new file in the Git repository to test the webhook

SET-UP SONARQUBE

Access the SonarQube instance from a new tab: ssh -i ssh-key-pair.pem ubuntu@SonarQube_EC2_public_IP
Update both instance hostnames:

>> JENKINS INSTANCE
sudo hostnamectl set-hostname jenkins
/bin/bash

>> SONARQUBE INSTANCE
sudo hostnamectl set-hostname sonarqube
/bin/bash

From the SonarQube instance run:

sudo apt update
sudo apt install openjdk-17-jdk (SonarQube commonly runs with JDK 17)
wget https://binaries.sonarsource.com/Distribution/sonarqube/sonarqube-9.9.4.87374.zip
ls
sudo apt install unzip
unzip sonarqube-9.9.4.87374.zip
ls
cd sonarqube-9.9.4.87374.zip/
ls
cd bin/
ls
cd linux-x86-64/
ls
./sonar.sh console (pass the console parameter to get the logs)

Open port 9000 for SonarQube's EC2 instance
Access SonarQube: http://100.27.210.125:9000
username: admin (default)
password: admin (default)

IMPORTANT:
>> Give your project a name, and select the respective CI (Jenkins) along with source code (GitHub). Copy the project key for later use.
>> Go to My Account > Security and create a global token, save for later.

CONNECT JENKINS TO SONARQUBE

Back to Jenkins, install the SonarQube Scan and SSH2 plugins
Once installed, go to Tools and scroll to the SonarScanner for MSBuild section, give it a name and hit Save
Navigate to System, scroll to SonarQube, give it a name and the respective URL: http://100.27.210.125:9000/
Now, go back to the Jenkins pipeline, click on Configure and under the Build steps section select "Execute SonarQube Scanner". Use Sonar's project key in the "Analysis properties" field.
In System settings, it's time to add a server authentication token for the Sonar-Server. The credentials type would be secret text. Use Sonar's global token from earlier in the secret field, give it any ID and save.
Test the pipeline

SET-UP DOCKER

Docker Installation Guide: https://docs.docker.com/engine/install/ubuntu/

Access Docker's EC2 instance via SSH following the same set of steps and update the hostname.

ssh -i ssh-key-pair.pem ubuntu@Docker_EC2_public_IP
sudo hostnamectl set-hostname docker-server
/bin/bash

NOTE: The -i flag specifies the identity (private key file) to the remote server.

------------------------------------------------------------------------------------------------------

CREATE A SECURE CONNECTION BETWEEN THE DOCKER AND THE JENKINS SERVERS

From the Jenkins server switch to the Jenkins user: sudo su Jenkins
From the Docker server switch to the root user: sudo su
Access the SSH config file: nano /etc/ssh/sshd_config, uncomment the PubkeyAuthentication yes line as well as the PasswordAuthentication yes one
NOTE: To exit the nano editor and save the changes, hit Ctrl + X, Y and ENTER
Restart the sshd service: systemctl restart ssh

ERROR: Permission denied (publickey).
TS: Sometimes cloud images override SSH settings in another file:

ls /etc/ssh/sshd_config.d
nano /etc/ssh/sshd_config.d/60-cloudimg-settings.conf
Changed "PasswordAuthentication" to yes
systemctl restart ssh
passwd ubuntu
**Create pwd**
ssh ubuntu@Docker_EC2_public_IP
exit

Generate a new SSH key:

ssh-keygen (keep everything to its default value)
ssh-copy-id ubuntu@Docker_EC2_public_IP

Back to Jenkins > System and scroll down to Server Groups Center, create a server group list, give it any name and enter the ubuntu username (default value) along with the created password
Add a Server List, use the name of the server group created, give a name to the server and use the public IP of the Docker's EC2 instance
Go to the automated-pipeline > Configure and add a new build step "Remote Shell", the target server should auto-populate, for the shell command do: touch test.txt

DEPLOY TO DOCKER

Documentation guide: https://hub.docker.com/_/nginx

Create a Dockerfile on GitHub
Create a website folder inside the Docker server: mkdir website
Change the ownership of the dir to be owned by the ubuntu user: sudo chown -R ubuntu:ubuntu website
Go back to Jenkins > automated-pipeline > Configure and replace the Remote Shell with an Execute Shell step: scp -r ./* ubuntu@3.91.69.132:~/website/
Create a user group:

sudo usermod -aG docker ubuntu
newgrp docker
docker ps

Add a Remote Shell step to the build stage. Here we'll create the image and run the container:

cd /home/ubuntu/website
docker build -t mywebsite .
docker run -d -p 8085:80 --name=Yitza-Website mywebsite

Open the port 8085 for the Docker instance.

WEBSITE ERROR: My index file lives inside the web-app dir, so I ran into a mapping issue in the Dockerfile, which made it default to NGINX's welcome page. To fix it, I changed the copy directive from COPY . /usr/share/nginx/html/ to:

FROM nginx
COPY ./web-app/ /usr/share/nginx/html/ (letting it know exactly where the index file was)

BUILD ERROR: docker: Error response from daemon: Conflict. The container name "/Yitza-Website" is already in use by container "9b033824295952a0f13e4603898a3d78766704ec4196f6bfce00606ea32879e4". You have to remove (or rename) that container to be able to reuse that name.

FIX: I had to add a related operator to the remote shell asking it to stop and remove any container with a matching name before attempting to start a new one:

cd /home/ubuntu/website

docker build -t mywebsite .

docker stop Yitza-Website || true
docker rm Yitza-Website || true

docker run -d -p 8085:80 --name Yitza-Website mywebsite
